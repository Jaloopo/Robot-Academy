# A11y- & UX-granskning – Edison Hemguide

Strukturerad genomgång av `docs/design.md` mot bygget (`js/app.js` + `style.css`), med
fokus på det som tillkommit sedan förra granskningen: `Klart`/`Påbörjat`-badges
(`.chapter-status`, `.rail-status`), bild-stegtypen (`.step-image` + `assets/edison-knappar.svg`)
och "Börja om kapitlet" (`.linklike`). Pixel-/visuell slutkoll sker i webbläsare (`file://`)
och är ägarens steg; det här är en kod-/spec-granskning + a11y-resonemang.

Senast granskad: 2026-06-20 (spår B).

`--color-edison-orange-dark` (`#C8540F`). `#F26B1D` är fortsatt ok för kant/fokusring/maskot
(dekor som inte bär text).

## Text bär betydelsen (inte enbart färg)
- ✓ Badges har texten `Klart`/`Påbörjat` – inte bara grön/amber yta.
- ✓ Rätt/fel-alternativ har ikon (`✓` / `↻`-SVG, `aria-hidden`) + textfeedback, inte bara färg.
- ✓ Bild-steg: `step.text` bär betydelsen, bilden är komplement med obligatorisk `alt`.
- ✓ Knappar i innehållet beskrivs med form + funktion ("triangel-knappen (play)").

## Fokus & tangentbord
- ✓ Synlig fokusring (`:focus-visible`, 2 px orange + offset) på knappar, länkar, alternativ,
  `.linklike`, rail-länkar och `#step-card`.
- ✓ Alla interaktiva element är riktiga `<button>`/`<a>` → nåbara med tangentbord.
- ✓ Efter rätt svar flyttas fokus till "Nästa" (`applyFocus("next")`); efter fel till första
  aktiva alternativet; vid stegbyte till `#step-card` (tabindex=-1) – logisk fokusordning.
- ✓ Avstängd "Nästa" kompletteras av synlig text `.nav-hint` ("Svara först för att gå vidare").

## Touch-mål (≥44 px)
- ✓ `.btn`, `.option`, `.chapter-link`, `.linklike` har `min-height: var(--touch-min)` (44 px).
- ✓ `.btn--ghost` ("Börja om" i ordningssteg) ärver 44 px från `.btn`.
- (i) `.rail-link`, `.rail-overview`, `.chapter-finish a` är desktop-/musmål (≥900 px) – inga
  touch-krav, men har god klickyta.

## Responsivitet (mobil → 720 px → rail 900 px)
- ✓ Allt desktop är additivt (`@media (min-width: 720px)` + `900px`); mobilens värden orörda.
- ✓ Under 900 px är `.chapter-rail { display: none }` → en kolumn, ingen dubblerad nav.
- ✓ `.step-image` är responsiv (`width:100%`, `max-width:320px`, `height:auto`).
- ✓ `.step-main { min-width: 0 }` låter grid-spalten krympa korrekt.

## Rörelse (prefers-reduced-motion)
- ✓ Maskot-nudge stängs av vid `prefers-reduced-motion: reduce`.
- **Åtgärd gjord:** även `transition` på `.progressbar > span`, `.btn`, `.option`,
  `.chapter-link`, `.rail-link` stängs nu av under reduced-motion (förut bara maskoten).

## Semantik / skärmläsare
- ✓ `lang="sv"`, en `<h1>` per vy, `<ol>`/`<li>` för kapitel- och stegrelaterade listor.
- ✓ `role="status" aria-live="polite"` på feedback och landningsmeddelande.
- ✓ `aria-current="step"` på aktivt rail-kapitel; `aria-pressed` på val; `aria-label` på
  ordningsval ("Vald som nummer X: …"); dekorativa ikoner/maskot `aria-hidden`.

## Noterat (ej åtgärdat – medvetet)
- (i) **Vilolägets kanter** på `.option`/`.chapter-link` (`#e0dbd3` mot vitt, ~1,2:1) når inte
  3:1 (WCAG 1.4.11 icke-text). Affordansen förstärks dock av hover/`:focus-visible` (orange,
  ≥3:1) och elementen innehåller text. Lämnas som lugn-design-avvägning; kandidat att skärpa
  om en framtida granskning vill ha tydligare vilolägeskant.
- (i) `aria-live`-paragrafen byggs om vid varje `render()`; eftersom hela `#app` ersätts kan
  uppläsning vara browserberoende. Fokusflytten till `#step-card`/"Nästa" mildrar detta.

- Visuell/pixel-koll i Chrome (`file://`) på mobilbredd, 720 px och ≥900 px är ägarens steg.
